# Contributing to BotDash

First of all, thank you for taking the time to contribute to this absolute monument of rage and PHP spaghetti. Your help is genuinely appreciated, even if it means you have to endure reading my code.

## How to Contribute

### 🐛 Bug Reports
If you find a bug, please open an Issue. But please, for the love of sanity, do not just write "it's broken." 
Provide:
- What you were trying to do.
- What actually happened.
- Steps to reproduce the issue.
- Any error logs (like from `webhook_log.txt`, `moderation_debug.txt` or PHP error logs).

### 💡 Feature Requests
Have a great idea? Open an Issue. Explain *why* this feature is useful for Kick streamers, not just *what* it does. If your idea requires API endpoints that Kick doesn't currently provide, well, we can't do magic.

### 🛠️ Pull Requests (PRs)
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, test it on a live Kick channel/webhook environment first.
3. Ensure your code matches the existing style. We use raw PHP, PDO, and vanilla JavaScript. No massive frameworks, no React/Vue overwrites, no unnecessary npm dependencies. Keep it raw, keep it vanilla.
4. Write a descriptive commit message.
5. Submit the PR and wait for the review.

## Nasıl Katkıda Bulunabilirsiniz?

### 🐛 Hata Bildirimleri (Bug Reports)
Bir hata bulursanız lütfen bir Issue (Sorun) açın. Ama lütfen sadece "çalışmıyor" yazıp bırakmayın.
Şunları eklemeyi unutmayın:
- Ne yapmaya çalışıyordunuz?
- Gerçekte ne oldu?
- Hatayı yeniden üretme adımları.
- Varsa hata logları (`webhook_log.txt`, `moderation_debug.txt` veya sunucu PHP logları).

### 💡 Özellik İstekleri (Feature Requests)
Harika bir fikriniz mi var? Bir Issue açın ve ne istediğinizi değil, bunun Kick yayıncıları için *neden* faydalı olacağını açıklayın. Eğer Kick'in henüz sağlamadığı (örneğin API'de olmayan) bir özellik gerektiriyorsa, maalesef sihir yapamıyoruz.

### 🛠️ Pull Request (PR) Göndermek
1. Projeyi fork'layın ve `main` dalı üzerinden kendi dalınızı (branch) oluşturun.
2. Yeni bir kod eklediyseniz, çalıştığından emin olmak için canlı bir Kick/Webhook ortamında test edin.
3. Mevcut kod stiline sadık kalın. Saf (raw) PHP, PDO ve standart JavaScript kullanıyoruz. Devasa framework'ler, React/Vue entegrasyonları veya gereksiz paketler (dependency) eklemeyin. Orijinal ruhuna sadık kalın.
4. Yaptığınız değişikliği anlatan düzgün bir commit mesajı yazın.
5. PR'ı gönderin ve incelemeyi bekleyin.
